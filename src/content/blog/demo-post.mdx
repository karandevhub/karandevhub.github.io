# Welcome to the New Blog

Building a portfolio is one thing, but making it **fast**, **SEO-friendly**, and **easy to maintain** is another. That's where Next.js and MDX come in.

## Why MDX?

MDX allows you to use JSX in your markdown content. You can import components, such as interactive charts or alerts, and embed them within your document. This makes writing long-form content with interactive elements a breeze.

> "MDX is an authorable format that lets you seamlessly write JSX in your Markdown documents."

## Code Highlighting

Syntax highlighting is built right in. Here's an example of how easy it is to fetch a post in Next.js:

```tsx
import { getAllPosts } from "@/lib/blog";

export default async function BlogPage() {
  const posts = getAllPosts();
  
  return (
    <div>
      {posts.map(post => (
        <h2 key={post.slug}>{post.title}</h2>
      ))}
    </div>
  );
}
```

## The Power of Static Site Generation (SSG)

Because this portfolio uses `output: 'export'`, every blog post you write is converted into a highly optimized, static HTML file. This means:
- **Zero server latency**: Files are served from a CDN.
- **Perfect SEO**: Google can instantly index the content.
- **Maximum Security**: No database means no SQL injections.
